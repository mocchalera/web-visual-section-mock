# references/01_web_section_patterns.md

## 目的

WEBページやLPを、意味のあるセクション単位に分解し、各セクションを1枚の完成した画像として設計するためのリファレンス。

## 共通ルール

- 1画像につき1セクションのみ
- セクションの途中で見切らない
- 各セクションは単体でも美しく成立させる
- 複数セクションを1枚に連結しない
- 情報が多い場合は縮小せず分割する
- 迷った場合は余白を取る
- 各セクションの主メッセージは1つに絞る

## セクション設計フォーマット

```text
section_id:
section_role:
main_message:
visible_text:
main_visual:
layout_pattern:
graphic_pattern:
typography_pattern:
color_lighting_direction:
spacing_policy:
split_decision:
avoid:
```

## 01. Hero Section

### 目的

最初の3秒で、ブランドの世界観・約束・期待値を伝える。ページ全体の印象を決定する最重要セクション。

### 入れる要素

- ブランド名または企画名
- メインコピー
- 短いサブコピー
- 主役ビジュアル
- CTA
- 小さな英字ラベル
- 大胆な余白

### 入れない要素

- 詳細な特徴一覧
- 料金表
- FAQ
- 長い説明文
- 実績の羅列
- 複数CTA
- 小さな注釈の大量配置

### 向いている構図

- 巨大タイポ × 主役ビジュアル
- Split Hero
- Magazine Cover Hero
- Poster Hero
- White Space Luxury
- Product Silhouette Hero

### 向いているタイポグラフィ

- 極太ゴシックロックアップ
- 漢字巨大化 × かな抑制
- 縦横ミックス
- アウトライン文字
- タイトルのロゴ化
- 欧文ラベル添え

### 分割判断

サブコピーが2行を超える、特徴を3つ以上入れたくなる、CTA以外の導線が増える場合は次セクションへ逃がす。

### 避ける

- LPのファーストビュー風テンプレート
- 普通のビジネス服で立っているだけの人物
- 白背景に黒文字を置いただけ
- ヒーローで全説明を済ませること

## 02. Problem Section

### 目的

読者の課題を自分ごと化する。不安を煽りすぎず、違和感や引っかかりで「自分のことかも」と思わせる。

### 入れる要素

- 短い課題提起コピー
- 課題を象徴するビジュアル
- 最小限の補足
- 余白
- 視線を止める小さな違和感

### 入れない要素

- 解決策の説明を全部入れる
- 長い箇条書き
- 不安を過剰に煽る表現
- 暗いトーン
- ネガティブすぎる人物写真

### 向いている構図

- 大胆な余白
- ズレたグリッド
- 反復構図
- Before / After Layout
- Editorial Grid
- フレームインフレーム

### 向いているタイポグラフィ

- 巨大一文字
- 助詞だけ小さくする
- 縦横ミックス
- 字間広めのサブコピー
- 短歌・俳句的余白

### 分割判断

課題が複数ある場合は、problem_01 / problem_02 / problem_03 のように分ける。

### 避ける

- 「こんなお悩みありませんか？」のテンプレ感
- チェックリストの詰め込み
- 怪しい情報商材風
- 暗い顔の人物だけで恐怖訴求すること

## 03. Benefit Section

### 目的

読者が得られる良い変化を、明るく直感的に提示する。「機能」ではなく「生活・仕事・感情がどう変わるか」を見せる。

### 入れる要素

- ベネフィット1〜3点
- 明るい変化を感じるビジュアル
- 短いコピー
- 必要なら小さなUIカード
- ポイントカラー

### 入れない要素

- 機能説明の羅列
- 数字やスペックの詰め込み
- 4つ以上のカード
- 長文の補足

### 向いている構図

- Floating Card Layout
- 色面グラフィック融合
- ハイキー色面
- Asymmetric Grid
- Card Stack Layout

### 向いているタイポグラフィ

- 極太ゴシックロックアップ
- 漢字巨大化 × かな抑制
- 欧文ラベル添え
- 2段ロックアップ

### 分割判断

ベネフィットが4つ以上ある場合は、overview / benefit_01 / benefit_02 に分ける。

### 避ける

- アイコン3つと説明文だけの凡庸な構成
- 読めないほど小さいカード
- 「便利」「簡単」「安心」だけで終わる表現

## 04. Features Section

### 目的

サービスや商品の特徴を、理解しやすく魅力的に見せる。「多機能」ではなく、選ぶ理由を整理する。

### 入れる要素

- 特徴1〜3点
- 機能を象徴するビジュアル
- 短いラベル
- 必要最小限の説明
- 比較やUIカード

### 入れない要素

- 6項目以上の特徴
- 長文説明
- 細かいスペック
- 似たようなカードの羅列

### 向いている構図

- Card Stack Layout
- Editorial Grid
- Asymmetric Grid
- Layered UI Hero
- Comparison Layout
- Timeline Layout

### 向いているタイポグラフィ

- 字間広めのサブコピー
- 欧文ラベル添え
- 長体ゴシック
- タイトルロックアップ

### 分割判断

特徴が多い場合は、features_overview / feature_01 / feature_02 のように分割する。

### 避ける

- 機能紹介スライドのような平板さ
- UIカードを小さく大量配置
- すべて同じサイズの見出し
- 説明文が主役になること

## 05. Process Section

### 目的

利用手順・導入ステップ・変化の流れを直感的に理解させる。

### 入れる要素

- 3ステップ程度の流れ
- 数字ラベル
- 短い見出し
- 動線を示す装飾
- シンプルな図形やカード

### 入れない要素

- 5ステップ以上を1枚に詰め込む
- 詳細な手続き説明
- 長い注意事項
- 小さすぎる矢印や注釈

### 向いている構図

- Timeline Layout
- Horizontal Scroll Inspired
- Card Stack Layout
- 反復構図
- 帯グラフィック

### 向いているタイポグラフィ

- 数字巨大化
- 欧文ラベル添え
- 長体ゴシック
- 縦長タイトル帯

### 分割判断

手順が4つ以上ある場合は、前半・後半に分ける。

### 避ける

- 説明書のような硬さ
- 矢印だらけの図解
- 情報量で安心させようとすること

## 06. Case Study Section

### 目的

実績・事例・導入後の変化を、信頼感とストーリーで見せる。

### 入れる要素

- 1つの事例
- 主要な成果
- 短い引用
- 写真または象徴ビジュアル
- 数字は最大1〜3個

### 入れない要素

- 複数事例の詰め込み
- 長いインタビュー本文
- 細かい実績数字
- 顔写真と表の大量配置

### 向いている構図

- Magazine Feature Layout
- Editorial Grid
- Full Bleed Photo
- Testimonial Card
- Frame in Frame

### 向いているタイポグラフィ

- 雑誌見出し風ジャンプ率
- 引用符を大きく扱う
- 字間広めのサブコピー
- 縦横ミックス

### 分割判断

複数事例を紹介する場合は、1事例1画像に分ける。

### 避ける

- 実績ロゴを大量に並べるだけ
- 数字を盛りすぎる
- お客様の声テンプレート感

---

## 07. Pricing Section

### 目的

料金やプランを、安心感と選びやすさを持って提示する。

### 入れる要素

- 主要プラン
- 価格
- 最小限の特徴
- 推奨プランの強調
- CTA

### 入れない要素

- すべての機能比較
- 小さな注釈の大量配置
- 4プラン以上の横並び
- 複雑な料金条件

### 向いている構図

- Comparison Layout
- Card Stack Layout
- Centered Layout
- White Space Luxury
- UI Card Layout

### 向いているタイポグラフィ

- 数字の巨大化
- 極太ゴシック
- 欧文ラベル添え
- 字間広めの補足

### 分割判断

料金が複雑な場合は、pricing_overview / pricing_detail に分ける。

### 避ける

- 比較表を無理に1枚に入れる
- 価格が読めない
- 注釈だらけで不安になる構成

---

## 08. Testimonial Section

### 目的

利用者の声を、信頼感・親近感・憧れとして伝える。

### 入れる要素

- 1〜3件の声
- 短い引用
- 写真またはアイコン
- 職業や属性
- 軽い装飾

### 入れない要素

- 長文レビュー
- 5件以上の口コミ
- 星評価だけ
- 小さいカードの大量配置

### 向いている構図

- Testimonial Wall
- Magazine Interview
- Card Stack Layout
- Editorial Grid
- Floating Quote Layout

### 向いているタイポグラフィ

- 引用符の巨大化
- 字間広めのサブコピー
- 縦横ミックス
- 欧文ラベル添え

### 分割判断

声が多い場合は、代表1件を大きく見せ、残りは別セクションに分割する。

### 避ける

- よくある口コミLP風
- 顔写真を小さく並べるだけ
- 星5評価を強調しすぎること

---

## 09. FAQ Section

### 目的

購入・申し込み前の不安を取り除く。ただし、画像化する場合はFAQ全量を入れない。

### 入れる要素

- 代表的な質問1〜3個
- 短い回答
- 整った余白
- 見出し
- 必要ならアコーディオン風UI

### 入れない要素

- 10個以上のFAQ
- 長文回答
- 細かい条件説明
- 規約のような文章

### 向いている構図

- Minimal FAQ Layout
- Accordion UI Layout
- White Space Luxury
- Editorial Grid

### 向いているタイポグラフィ

- Qの巨大化
- 字間広めの補足
- 極太ゴシック見出し
- 欧文ラベル添え

### 分割判断

質問が多い場合は、faq_basic / faq_payment / faq_support のようにカテゴリごとに分割する。

### 避ける

- FAQ全部を詰め込む
- 文字が小さい
- セクションとして地味すぎる

---

## 10. CTA Section

### 目的

読者に次の行動を促す。最後に感情をもう一度高め、申し込みや問い合わせにつなげる。

### 入れる要素

- 強い短文コピー
- CTAボタン
- ブランド名
- 最小限の補足
- 主役ビジュアルまたは象徴的な色面
- 大胆な余白

### 入れない要素

- 詳細説明
- 複数の導線
- 大量の特典
- 不安を煽る文言

### 向いている構図

- CTA Poster Layout
- 巨大タイポ衝突
- White Space Luxury
- Centered Poster
- 色面分割

### 向いているタイポグラフィ

- 極太ゴシックロックアップ
- 巨大一文字
- 漢字巨大化
- 2段ロックアップ
- アウトライン文字

### 分割判断

CTAにキャンペーン情報や特典を入れたい場合は、CTA本体と補足セクションに分ける。

### 避ける

- 「今すぐ申し込む」だけの安い見せ方
- ボタンを複数並べる
- 情報商材っぽい煽り

---

## 11. Footer Section

### 目的

ページの終わりとして、ブランドの余韻と信頼感を残す。

### 入れる要素

- ブランド名
- 最小限のリンク風情報
- コピーライト風表記
- 静かな余白
- 小さなアクセント

### 入れない要素

- 大量リンク
- 細かい住所や規約
- 複雑なサイトマップ
- フッターだけで説明を増やすこと

### 向いている構図

- Minimal Footer Layout
- White Space Luxury
- Small Logo Layout
- Editorial End Page

### 向いているタイポグラフィ

- 字間広め
- 小さな欧文
- 余白重視
- ブランドロゴ風文字

### 分割判断

フッターは説明を増やす場所ではなく、終わりの余韻を作る場所として扱う。

### 避ける

- CMSテンプレートのようなフッター
- 情報密度が高すぎる
- 終わりの余韻がない
