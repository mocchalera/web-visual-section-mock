# references/00_reference_index.md

## 目的

このリファレンス群は、WEBビジュアル・指針モックエージェントが、WEBページやLPのビジュアルを**セクション単位の高品質画像**として生成するための型ライブラリです。

本体プロンプトには、役割・最重要制約・出力ルールだけを置き、詳細なデザイン判断はこのリファレンス群を参照して行います。

## 最重要ルール

- WEBページ全体を1枚画像にまとめない
- 1画像につき1セクションのみ
- セクション途中で見切らない
- アスペクト比に合わせて詰め込まない
- 入りきらない場合は分割する
- 迷った場合は余白を選ぶ
- 各セクションでは、使う型を少数に絞る

## 使用順序

```text
1. WEBページ全体か、単体ビジュアルかを判断する
2. WEBページならセクション単位に分割する
3. 各セクションの役割と主メッセージを決める
4. 参考カンプや実装前提がある場合はレイヤー、幅、素材分解を行う
5. 人物写真を強い既定候補としつつ、必要なら意図的に別モチーフを主役にする
6. WEBセクションパターンを参照する
7. WEBレイアウトパターンを1つ選ぶ
8. グラフィック構図パターンを1つ選ぶ
9. 日本語タイポグラフィパターンを1〜2つ選ぶ
10. 色・光・写真・ファッションの方向性を選ぶ
11. 業種別トーンでありがちな失敗を避ける
12. 画像生成プロンプトへ変換する
13. NGチェックを行う
```

## 型の選択制限

```text
WEBレイアウト型: 1つ
グラフィック構図型: 1つ
日本語タイポグラフィ型: 1〜2つ
主役ビジュアル型: 1つ
写真・光・色彩型: 1つ
業種別トーン: 1つ
```

参照パターンは材料であり、すべて使ってはいけません。複数の型を無理に混ぜると、情報過多、安っぽさ、視線の迷子、AIっぽさが発生します。

## 判断の優先順位

```text
1. セクションが途中で見切れないこと
2. 1画像1セクションであること
3. 1画面1メッセージであること
4. スマホで読めること
5. 余白があること
6. 日本語タイポグラフィが美しいこと
7. 主役ビジュアルがメッセージ由来であり、人物写真を使う場合も別モチーフにする場合も意図が明確なこと
8. 参考カンプがある場合、レイヤー順、全幅/コンテナ、素材分解が明確なこと
9. 写真・色・文字が同じ世界にあること
10. 広告として手が止まる違和感があること
```

## 運用メモ

- 本体プロンプトは短く保つ。
- リファレンスは「全部使うもの」ではなく「選ぶもの」。
- 各セクションごとに、最も効く型だけを選ぶ。
- 出力が崩れた場合は、まずセクション分割・見切れ・余白を直す。
- 迷ったら Person-led を初期候補にし、商品、UI、空間、タイポグラフィ、抽象グラフィック、データ表現の方がメッセージに強い場合だけ主役を切り替える。
- 参考カンプが複雑な場合は、先に `10_visual_decomposition_for_web_build.md` で「背景プレート、透過切り抜き、SVG/CSS装飾、実テキスト」を分けてからプロンプト化する。
