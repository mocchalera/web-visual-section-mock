# references/00_reference_index.md

## 目的

このリファレンス群は、WEBビジュアル・指針モックエージェントが、WEBページやLPのビジュアルを**セクション単位の高品質画像**として生成するための型ライブラリです。

本体プロンプトには、役割・最重要制約・出力ルールだけを置き、詳細なデザイン判断はこのリファレンス群を参照して行います。

## 最重要ルール

- WEBページ全体を1枚画像にまとめない
- 1画像につき1セクションのみ
- セクション途中で見切らない
- アスペクト比に合わせて詰め込まない
- 入りきらない場合は分割する
- 迷った場合は余白を選ぶ
- 各セクションでは、使う型を少数に絞る

## リファレンス一覧

```text
references/
  00_reference_index.md
  01_web_section_patterns.md
  02_web_layout_patterns.md
  03_graphic_composition_patterns.md
  04_japanese_typography_patterns.md
  05_color_lighting_fashion_patterns.md
  06_industry_tone_patterns.md
  07_negative_examples.md
  08_prompt_templates.md
  09_main_visual_archetypes.md
```

## 使用順序

```text
1. WEBページ全体か、単体ビジュアルかを判断する
2. WEBページならセクション単位に分割する
3. 各セクションの役割と主メッセージを決める
4. 人物写真を強い既定候補としつつ、必要なら意図的に別モチーフを主役にする
5. WEBセクションパターンを参照する
6. WEBレイアウトパターンを1つ選ぶ
7. グラフィック構図パターンを1つ選ぶ
8. 日本語タイポグラフィパターンを1〜2つ選ぶ
9. 色・光・写真・ファッションの方向性を選ぶ
10. 業種別トーンでありがちな失敗を避ける
11. 画像生成プロンプトへ変換する
12. NGチェックを行う
```

## 型の選択制限

```text
WEBレイアウト型: 1つ
グラフィック構図型: 1つ
日本語タイポグラフィ型: 1〜2つ
主役ビジュアル型: 1つ
写真・光・色彩型: 1つ
業種別トーン: 1つ
```

参照パターンは材料であり、すべて使ってはいけません。複数の型を無理に混ぜると、情報過多、安っぽさ、視線の迷子、AIっぽさが発生します。

## 判断の優先順位

```text
1. セクションが途中で見切れないこと
2. 1画像1セクションであること
3. 1画面1メッセージであること
4. スマホで読めること
5. 余白があること
6. 日本語タイポグラフィが美しいこと
7. 主役ビジュアルがメッセージ由来であり、人物写真を使う場合も別モチーフにする場合も意図が明確なこと
8. 写真・色・文字が同じ世界にあること
9. 広告として手が止まる違和感があること
```

## 運用メモ

- 本体プロンプトは短く保つ。
- リファレンスは「全部使うもの」ではなく「選ぶもの」。
- 各セクションごとに、最も効く型だけを選ぶ。
- 出力が崩れた場合は、まずセクション分割・見切れ・余白を直す。
- 迷ったら Person-led を初期候補にし、商品、UI、空間、タイポグラフィ、抽象グラフィック、データ表現の方がメッセージに強い場合だけ主役を切り替える。

# references/01_web_section_patterns.md

## 目的

WEBページやLPを、意味のあるセクション単位に分解し、各セクションを1枚の完成した画像として設計するためのリファレンス。

## 共通ルール

- 1画像につき1セクションのみ
- セクションの途中で見切らない
- 各セクションは単体でも美しく成立させる
- 複数セクションを1枚に連結しない
- 情報が多い場合は縮小せず分割する
- 迷った場合は余白を取る
- 各セクションの主メッセージは1つに絞る

## セクション設計フォーマット

```text
section_id:
section_role:
main_message:
visible_text:
main_visual:
layout_pattern:
graphic_pattern:
typography_pattern:
color_lighting_direction:
spacing_policy:
split_decision:
avoid:
```

## 01. Hero Section

### 目的

最初の3秒で、ブランドの世界観・約束・期待値を伝える。ページ全体の印象を決定する最重要セクション。

### 入れる要素

- ブランド名または企画名
- メインコピー
- 短いサブコピー
- 主役ビジュアル
- CTA
- 小さな英字ラベル
- 大胆な余白

### 入れない要素

- 詳細な特徴一覧
- 料金表
- FAQ
- 長い説明文
- 実績の羅列
- 複数CTA
- 小さな注釈の大量配置

### 向いている構図

- 巨大タイポ × 主役ビジュアル
- Split Hero
- Magazine Cover Hero
- Poster Hero
- White Space Luxury
- Product Silhouette Hero

### 向いているタイポグラフィ

- 極太ゴシックロックアップ
- 漢字巨大化 × かな抑制
- 縦横ミックス
- アウトライン文字
- タイトルのロゴ化
- 欧文ラベル添え

### 分割判断

サブコピーが2行を超える、特徴を3つ以上入れたくなる、CTA以外の導線が増える場合は次セクションへ逃がす。

### 避ける

- LPのファーストビュー風テンプレート
- 普通のビジネス服で立っているだけの人物
- 白背景に黒文字を置いただけ
- ヒーローで全説明を済ませること

---

## 02. Problem Section

### 目的

読者の課題を自分ごと化する。不安を煽りすぎず、違和感や引っかかりで「自分のことかも」と思わせる。

### 入れる要素

- 短い課題提起コピー
- 課題を象徴するビジュアル
- 最小限の補足
- 余白
- 視線を止める小さな違和感

### 入れない要素

- 解決策の説明を全部入れる
- 長い箇条書き
- 不安を過剰に煽る表現
- 暗いトーン
- ネガティブすぎる人物写真

### 向いている構図

- 大胆な余白
- ズレたグリッド
- 反復構図
- Before / After Layout
- Editorial Grid
- フレームインフレーム

### 向いているタイポグラフィ

- 巨大一文字
- 助詞だけ小さくする
- 縦横ミックス
- 字間広めのサブコピー
- 短歌・俳句的余白

### 分割判断

課題が複数ある場合は、problem_01 / problem_02 / problem_03 のように分ける。

### 避ける

- 「こんなお悩みありませんか？」のテンプレ感
- チェックリストの詰め込み
- 怪しい情報商材風
- 暗い顔の人物だけで恐怖訴求すること

---

## 03. Benefit Section

### 目的

読者が得られる良い変化を、明るく直感的に提示する。「機能」ではなく「生活・仕事・感情がどう変わるか」を見せる。

### 入れる要素

- ベネフィット1〜3点
- 明るい変化を感じるビジュアル
- 短いコピー
- 必要なら小さなUIカード
- ポイントカラー

### 入れない要素

- 機能説明の羅列
- 数字やスペックの詰め込み
- 4つ以上のカード
- 長文の補足

### 向いている構図

- Floating Card Layout
- 色面グラフィック融合
- ハイキー色面
- Asymmetric Grid
- Card Stack Layout

### 向いているタイポグラフィ

- 極太ゴシックロックアップ
- 漢字巨大化 × かな抑制
- 欧文ラベル添え
- 2段ロックアップ

### 分割判断

ベネフィットが4つ以上ある場合は、overview / benefit_01 / benefit_02 に分ける。

### 避ける

- アイコン3つと説明文だけの凡庸な構成
- 読めないほど小さいカード
- 「便利」「簡単」「安心」だけで終わる表現

---

## 04. Features Section

### 目的

サービスや商品の特徴を、理解しやすく魅力的に見せる。「多機能」ではなく、選ぶ理由を整理する。

### 入れる要素

- 特徴1〜3点
- 機能を象徴するビジュアル
- 短いラベル
- 必要最小限の説明
- 比較やUIカード

### 入れない要素

- 6項目以上の特徴
- 長文説明
- 細かいスペック
- 似たようなカードの羅列

### 向いている構図

- Card Stack Layout
- Editorial Grid
- Asymmetric Grid
- Layered UI Hero
- Comparison Layout
- Timeline Layout

### 向いているタイポグラフィ

- 字間広めのサブコピー
- 欧文ラベル添え
- 長体ゴシック
- タイトルロックアップ

### 分割判断

特徴が多い場合は、features_overview / feature_01 / feature_02 のように分割する。

### 避ける

- 機能紹介スライドのような平板さ
- UIカードを小さく大量配置
- すべて同じサイズの見出し
- 説明文が主役になること

---

## 05. Process Section

### 目的

利用手順・導入ステップ・変化の流れを直感的に理解させる。

### 入れる要素

- 3ステップ程度の流れ
- 数字ラベル
- 短い見出し
- 動線を示す装飾
- シンプルな図形やカード

### 入れない要素

- 5ステップ以上を1枚に詰め込む
- 詳細な手続き説明
- 長い注意事項
- 小さすぎる矢印や注釈

### 向いている構図

- Timeline Layout
- Horizontal Scroll Inspired
- Card Stack Layout
- 反復構図
- 帯グラフィック

### 向いているタイポグラフィ

- 数字巨大化
- 欧文ラベル添え
- 長体ゴシック
- 縦長タイトル帯

### 分割判断

手順が4つ以上ある場合は、前半・後半に分ける。

### 避ける

- 説明書のような硬さ
- 矢印だらけの図解
- 情報量で安心させようとすること

---

## 06. Case Study Section

### 目的

実績・事例・導入後の変化を、信頼感とストーリーで見せる。

### 入れる要素

- 1つの事例
- 主要な成果
- 短い引用
- 写真または象徴ビジュアル
- 数字は最大1〜3個

### 入れない要素

- 複数事例の詰め込み
- 長いインタビュー本文
- 細かい実績数字
- 顔写真と表の大量配置

### 向いている構図

- Magazine Feature Layout
- Editorial Grid
- Full Bleed Photo
- Testimonial Card
- Frame in Frame

### 向いているタイポグラフィ

- 雑誌見出し風ジャンプ率
- 引用符を大きく扱う
- 字間広めのサブコピー
- 縦横ミックス

### 分割判断

複数事例を紹介する場合は、1事例1画像に分ける。

### 避ける

- 実績ロゴを大量に並べるだけ
- 数字を盛りすぎる
- お客様の声テンプレート感

### 入れる要素

- 主要プラン
- 価格
- 最小限の特徴
- 推奨プランの強調
- CTA

### 入れない要素

- すべての機能比較
- 小さな注釈の大量配置
- 4プラン以上の横並び
- 複雑な料金条件

### 向いている構図

- Comparison Layout
- Card Stack Layout

- White Space Luxury
- UI Card Layout

### 向いているタイポグラフィ

- 極太ゴシック

- 字間広めの補足

### 分割判断

- 文字が小さい
- セクションとして地味すぎる

# references/02_web_layout_patterns.md

## 目的

WEBセクション画像のレイアウト型を選ぶためのリファレンス。1セクションにつき、原則として1つのレイアウト型だけを選ぶ。

## 共通ルール

- レイアウトは情報整理のために使う
- 装飾ではなく、視線誘導を設計する
- 1枚に複数レイアウト型を混ぜない
- 余白を削ってまでカードや要素を増やさない
- スマホで見たときに主従が一瞬で伝わるようにする

---

## 01. Split Hero

### 特徴

画面を左右または上下に大きく分割し、片側にコピー、片側に主役ビジュアルを置く。

### 向いている用途

- Hero
- サービス紹介
- 個人ブランド
- SaaS
- 教育
- 商品LP

### 構成

- 左: メインコピー、サブコピー、CTA
- 右: 人物、商品、UI、象徴ビジュアル
- 境界に色面やラインを使う
- 片側は余白を大きく残す

### 避ける

- 左右両方に情報を詰める
- 両側の主張が同じ強さになる
- CTAやラベルを増やしすぎる

---

## 02. Centered Editorial Hero

### 特徴

中央にコピーや主役を置き、余白で高級感・信頼感を作る。

### 向いている用途

- 高級感のあるHero
- ブランド紹介
- CTA
- コンサルティング
- 個人ブランド

### 構成

- 中央に短いコピー
- 上下左右に大きな余白
- 小さなブランド名
- 背景は明るく静か
- 装飾は最小限

### 避ける

- 中央に長文を置く
- 余白を埋めようとする
- ボタンやバッジを増やす

---

## 03. Magazine Cover Hero

### 特徴

雑誌表紙のように、写真とタイポグラフィを強く組み合わせる。

### 向いている用途

- Hero
- キャンペーン
- クリエイター向けサービス
- ファッション
- 採用
- イベント

### 構成

- 人物または商品を大きく配置
- メインコピーを表紙タイトルのように扱う
- サブコピーを縦組みや小見出しとして配置
- 余白と情報量のバランスを取る

### 避ける

- 文字を大量に置きすぎる
- 雑誌表紙の模倣に見える
- 顔や商品を文字で潰す

---

## 04. Poster Hero

### 特徴

広告ポスターのように、1メッセージを強く打ち出す。

### 向いている用途

- Hero
- CTA
- キャンペーン
- 新商品告知
- イベント告知

### 構成

- コピーを大きく
- 主役ビジュアルを大胆に
- CTAは小さく明確に
- 色面や余白で視線を止める

### 避ける

- LP的な説明を入れる
- 複数の訴求を混ぜる
- 情報商材バナー風になる

---

## 05. Product Silhouette Hero

### 特徴

商品やロゴの形を大きなシルエット・抜き型・背景フレームとして使う。

### 向いている用途

- 商品LP
- ブランドサイト
- アプリ
- プロダクト紹介
- ロゴの印象を残したい案件

### 構成

- 商品シルエットを巨大化
- テキストを商品形状に沿わせる
- 背景色と商品色を連動
- アクセントカラーを一点だけ効かせる

### 避ける

- 商品が小さくなる
- ロゴをただ置くだけ
- シルエットが意味不明になる

---

## 06. Layered UI Hero

### 特徴

アプリ画面、WEB画面、カードUIをレイヤー状に重ねて、機能性と世界観を見せる。

### 向いている用途

- SaaS
- AIサービス
- アプリ
- ダッシュボード

# references/03_graphic_composition_patterns.md

## 目的

WEBセクション画像に、広告・雑誌・ポスターのような「スマホで手が止まる違和感」を与えるためのグラフィック構図リファレンス。

## 共通ルール

- 1セクションにつき主構図は1つだけ選ぶ
- 違和感は小さくてもよい
- 読ませる情報は欠けさせない
- 装飾のための装飾にしない
- 余白を埋める目的で使わない
- グラフィック構図はメッセージを強めるために使う

---

## 01. 巨大タイポ衝突

### 効果

コピーがロゴのように記憶に残る。文字と人物・商品が衝突することで視線が止まる。

### 向いている用途

- Hero
- CTA
- キャンペーン
- ブランドメッセージ
- イベント告知

### 設計

- メインコピーを画面の30〜60%まで大きくする
- 人物や商品と一部重ねる
- 重要な文字は欠けさせない
- 装飾的な一部文字のみ裁ち落とし可
- 文字を背景ではなく主役として扱う

### 避ける

- 全文を巨大化する
- 読めないほど崩す
- 余白を消す
- 安いバナーのような太字だけの表現

---

## 02. 余白一点突破

### 効果

情報を減らすことで、品・信頼・高級感を出す。スマホ画面で逆に目立つ。

### 向いている用途

- Hero
- CTA
- 高価格サービス
- ブランド紹介
- 個人ブランド

### 設計

- 要素を1〜3個に限定
- コピーを短くする
- 主役ビジュアルを小さく置いてもよい
- 余白を恐れない
- アクセントカラーを一点だけ使う

### 避ける

- 未完成に見えるほど弱い配置
- 文字が小さすぎる
- 余白を装飾で埋める

---

## 03. 色面分割

### 効果

画面に鮮度と構造を作る。セクションの意味やBefore / Afterを直感化できる。

### 向いている用途

- Hero
- Benefit
- Problem
- Before / After
- Features

### 設計

- 2〜3色の大きな色面で画面を分ける
- 片方にコピー、片方に主役
- 境界を直線・斜め・曲線で設計
- アクセントカラーを小さく効かせる

### 避ける

- 色数を増やしすぎる
- 背景が主役を邪魔する
- 原色だけで下品になる

---

## 04. 斜め構図

### 効果

静止画に動き・勢い・現代感を出す。

### 向いている用途

- キャンペーン
- イベント
- 採用
- 若年層向けサービス
- 新商品告知

### 設計

- 色面や文字ブロックを斜めに配置
- 主役の視線や動作と連動させる
- 傾ける要素は1〜2個に絞る
- 文字の可読性を守る

### 避ける

- 全要素を傾ける
- 読みにくくなる
- 古いスポーツチラシ風になる

---

## 05. 反復構図

### 効果

リズム・記憶・キャンペーン感を作る。同じ形や言葉を繰り返すことで印象が強まる。

### 向いている用途

- Features
- Product Lineup
- Campaign
- Problem
- Testimonial

### 設計

- 1つだけズラして違和感を作る
- サイズや色を少し変える
- 主役を埋もれさせない

### 避ける

- 単調なコピペに見える
- 情報カードを増やしすぎる
- すべて同じ強さになる

## 目的

日本語を単なる説明テキストではなく、WEBビジュアル・広告ビジュアルの主役として設計するためのリファレンス。

## 共通ルール

- 1セクションにつき主タイポグラフィ型は1つ
- 補助型は最大2つまで
- 読ませる文字は欠けさせない
- 装飾的な裁ち落としは許可

- コピーが長い場合は短く編集する
- 迷ったらジャンプ率を大きくし、余白を取る
- 日本語の助詞・かな・漢字のサイズ差を活かす

## コピー編集ルール

### メインコピー

- 短くする
- 1行または2行を基本にする
- 意味の核になる言葉を前に出す
- 曖昧な美辞麗句を避ける
- 読者の感情に刺さる動詞を入れる

### サブコピー

- メインコピーを説明しすぎない
- 1〜2行まで
- 文字間を広く取って品を出す
- 小さくしすぎない

### 避けるコピー

```text
未来を変える
あなたらしく輝く
新しい時代へ
可能性を広げる
すべての人に寄り添う
理想を叶える
```

使う場合は、より具体的な言葉に変換する。

## 01. 極太ゴシックロックアップ

### 用途

Hero、CTA、キャンペーン、強いブランドコピー。

### 特徴

短い日本語コピーを、ロゴのように一体化して見せる。

### 設計

- メイン語句を極太にする
- 助詞や補足語を小さくする
- 行間を詰めすぎない
- 文字ブロック全体を1つの図形として扱う
- 余白を大胆に取る

### 避ける

- 長文に使う
- 全文字を同じ大きさにする
- 文字を画面いっぱいに詰める

---

## 02. 長体ゴシック

### 用途

都会的、モード、シャープ、知的な印象を出したいとき。

### 特徴

文字を縦に細長く見せ、緊張感と洗練を出す。

### 設計

- 見出しに使う
- 字間はやや詰める
- 余白を大きく取る
- 欧文ラベルと相性が良い

### 避ける

- 本文に使う
- 長文を長体化する
- 可読性を犠牲にする

---

## 03. 平体コピー

### 用途

ポップ、親しみ、少しクセのある雰囲気。

### 特徴

文字を横に広げ、グラフィック感を出す。

### 設計

- 短い言葉に限定
- ポイントカラーと組み合わせる
- ラベルやステッカーと相性が良い

### 避ける

- 品を出したい場面で使いすぎる
- 全文を平体にする
- チラシ感が出る

---

## 04. 巨大一文字

### 用途

Problem、Hero、CTA、和モダン、強い印象を作る場面。

### 特徴

意味の核となる漢字一文字を巨大化し、画面の象徴にする。

### 設計

- 画面の30〜70%に配置
- 背景や人物と重ねる
- 補足テキストは小さく
- 装飾的に一部裁ち落とし可

### 避ける

- 意味の弱い文字を選ぶ
- 読ませる文字まで欠ける
- ただ大きいだけになる

---

## 05. 漢字巨大化 × かな抑制

### 用途

日本語らしい強さと品を出したい広告見出し。

### 特徴

意味の核になる漢字を大きくし、助詞・かなを小さくしてリズムを作る。

### 設計

- 漢字を大きく
- かなや助詞を小さく
- 行間は詰めすぎない
- 1〜2語を主役化する

### 避ける

- 全部を同じサイズにする
- 助詞まで大きくする
- 長文で使う

---

## 06. 助詞だけ小さくする

### 用途

コピーに広告的なリズムを作りたいとき。

### 特徴

「の」「を」「に」「が」「で」などを小さくして、主語や動詞を強調する。

### 設計

- 助詞を50〜70%程度に見せる
- ベースラインを揃えすぎない
- 文字ブロックとして美しく整える

### 避ける

- 助詞が読めない
- ばらばらに見える
- 意味が分断される

---

## 07. 縦横ミックス

### 用途

雑誌的、和モダン、知的、違和感を出したいとき。

### 特徴

縦組みと横組みを組み合わせ、視線にリズムを作る。

### 設計

- メインコピーは横組み
- サブコピーやラベルを縦組みにする
- 縦組みは画面端や余白に置く
- 主役ビジュアルと重ねすぎない

### 避ける

- 全部を縦横混在にする
- 小さい文字を縦組みにする
- 可読性を落とす

---

## 08. 欧文ラベル添え

### 用途

WEBらしさ、編集感、ブランド感を足す。

### 特徴

小さな英字を添えて、画面に構造と品を与える。

### 設計

- FEATURES / CASE STUDY / NEW STANDARD などを小さく置く
- 字間を広げる
- 日本語コピーの補助にする
- 意味不明な英字は禁止

### 避ける

- 英字が主役になる
- 意味のない英語を置く
- スペルミス風のAI文字

---

## 09. 字間広めのサブコピー

### 用途

品、余白、呼吸感を作りたいとき。

### 特徴

サブコピーの字間を広く見せ、静かな高品質感を出す。

### 設計

- 短文に限定
- 行間を広めに
- メインコピーより明らかに小さく
- 余白の中に置く

### 避ける

- 長文に使う
- 読めないほど字間を広げる
- メインコピーの力を奪う

---

## 10. 縦長タイトル帯

### 用途

雑誌風、和モダン、セクションタイトル。

### 特徴

縦方向の帯にタイトルを収め、画面の軸を作る。

### 設計

- 画面端に細い帯
- 縦組みまたは長体文字
- 色面と組み合わせる
- 主役ビジュアルを邪魔しない

### 避ける

- 帯が太すぎる
- 本文まで帯に入れる
- 古い和風チラシ感

---

## 11. アウトライン文字

### 用途

背景に文字の存在感を出したいが、重くしたくないとき。

### 特徴

線だけの文字で、軽さと奥行きを作る。

### 設計

- 背景や装飾に使う
- メインコピーは塗り文字で読ませる
- アウトライン文字は大きく
- 読めなくても意味の補助なら可

### 避ける

- メインコピーを全部アウトラインにする
- 線が細すぎる
- 背景と同化する

---

## 12. 塗り文字 × 線文字

### 用途

ジャンプ率と視覚的な差を作る。

### 特徴

一部を塗り文字、一部をアウトライン文字にして、リズムを作る。

### 設計

- 意味の核は塗り文字
- 補助語や反復語は線文字
- 2種類以上増やさない
- 色と連動させる

### 避ける

- すべての単語で効果を使う
- 読めない
- 派手すぎる

---

## 13. 反復タイポ

### 用途

キャンペーン感、リズム、記憶定着。

### 特徴

同じ言葉や短いフレーズを繰り返し、グラフィック化する。

### 設計

- 2〜5回程度
- 1つだけ色やサイズを変える
- 背景として使ってもよい
- 主コピーを別に用意する

### 避ける

- しつこい
- 読ませるコピーが埋もれる
- 意味がない反復

---

## 14. 画面端裁ち落とし

### 用途

迫力、広がり、ポスター感。

### 特徴

文字の一部を画面外に出し、画面外の広がりを感じさせる。

### 設計

- 装飾的な大文字に使う
- 読ませる本文には使わない
- 重要語の意味は残す
- 主役ビジュアルと連動させる

### 避ける

- 読ませる文字が欠ける
- セクション途中の見切れに見える
- トリミングミスに見える

---

## 15. 人物への食い込み

### 用途

写真と文字を一体化させたいとき。

### 特徴

文字が人物や商品に食い込み、広告的な緊張感を作る。

### 設計

- 肩、服、背景に重ねる
- 顔、目、手元、商品ロゴは隠さない
- 重なり部分に色面や影を少し入れる
- 読める範囲に留める

### 避ける

- 顔を隠す
- 読めない
- 合成が雑に見える

---

## 16. 商品を避ける文字組み

### 用途

商品を主役にしながら、文字も美しく配置したいとき。

### 特徴

商品形状に沿って文字を逃がし、自然な余白を作る。

### 設計

- 商品の輪郭を意識
- 文字ブロックを斜めや縦に逃がす
- 商品周辺に余白を作る
- CTAは独立して読ませる

### 避ける

- 商品に文字がかかりすぎる
- 文字が散らばる
- 読み順がわからない

---

## 17. 文字を図形化する

### 用途

タイポグラフィをビジュアル主役にしたいとき。

### 特徴

文字ブロックを四角、円、帯、シンボルのように扱う。

### 設計

- コピーを短くする
- 文字の外形を整える
- 行ごとの幅をコントロールする
- 図形や写真と組み合わせる

### 避ける

- 長文を無理に図形化
- 読みづらい
- 意味より形が勝つ

---

## 18. タイトルをロゴ化する

### 用途

ブランド名、企画名、サービス名を強く印象づけたいとき。

### 特徴

タイトルを既成フォントの文字列ではなく、ロゴのように設計する。

### 設計

- 字間を調整
- 一部の文字サイズを変える
- 欧文や記号を添える
- 色や線で一体化する

### 避ける

- ロゴを作り込みすぎて読めない
- 装飾過多
- 既存ブランドの模倣

---

## 19. コピーの2段ロックアップ

### 用途

短いコピーにリズムと構造を作りたいとき。

### 特徴

上下2段で意味を切り、視線の流れを作る。

### 設計

- 上段を小さく
- 下段を大きく
- 重要語を下段に置く
- 行間を意図的に設計

### 避ける

- 3段以上に増やす
- 全段同じ強さ
- コピーが長い

## 20. 短歌・俳句的余白

### 用途

静かで品のある印象、余韻、詩的な世界観。

### 特徴

短い言葉と大きな余白で、意味を読み手に委ねる。

### 設計

- 文字数を少なく
- 行間を広く
- 写真や色面は静かに
- CTAは小さく整える

### 避ける

- 何を伝えたいかわからない
- 抽象的すぎる
- 商業目的が消える

## 21. 雑誌見出し風ジャンプ率

### 用途

Case Study、Brand Story、採用、教育系。

### 特徴

見出し、リード、本文、ラベルの差を大きくし、編集的な読み心地を作る。

### 設計

- 見出しを大きく
- リードは中サイズ
- 補足は小さく
- 写真とグリッドで支える

### 避ける

- 全文同じサイズ
- 情報が雑誌記事のように多すぎる
- CTAが消える

# references/09_main_visual_archetypes.md

## 目的

WEBセクションの主役ビジュアルを、人物写真を強い既定候補として扱いながら、必要に応じて別モチーフへ意図的に切り替えるためのリファレンス。

主役ビジュアルの型は、クオリティを分散させるためではなく、**そのセクションの一番強い意味を何で見せるか**を決めるために使う。

## 最重要ルール

- 各セクションで主役ビジュアルの型を1つだけ選ぶ
- 人物写真は強い既定候補として扱う
- 特にHero、Benefit、CTA、採用、教育、個人ブランド、体験型サービスでは Person-led を初期候補にする
- 別モチーフは、人物よりも商品、UI、空間、物、抽象グラフィック、タイポグラフィ、データ表現の方がメッセージを鋭く見せられる場合に意図的に選ぶ
- 人物を使わないこと自体を目的にしない
- 複数の型を同格に並べない
- 補助ビジュアルは主役の意味を支えるだけにする
- 主役は大きく、明確に、余白を持って配置する
- どの型でも、広告品質の光、色、余白、タイポグラフィを必ず設計する

## 選択順

```text
1. このセクションの主メッセージは何か
2. まず Person-led を初期候補に置く
3. そのメッセージは、人・物・画面・空間・構造・言葉のどれで最も強く伝わるか
4. 人物より強い理由がある場合だけ、別モチーフへ切り替える
5. 主役ビジュアルの型を1つ選ぶ
6. 色・光・タイポグラフィを、その型に合わせて統一する
```

迷った場合は、Person-led を初期候補にする。ただし、商品・UI・空間・タイポグラフィ・抽象グラフィックの方が一瞬で価値を伝えられる場合は、別モチーフを主役にする。

# 主役ビジュアルの型

## 01. Person-led

### 向いている用途

- 個人ブランド
- 採用
- スクール
- コーチング
- 体験型サービス
- 顧客の感情変化を見せたいセクション
- Hero
- Benefit
- CTA

### 使う条件

人物の表情、姿勢、スタイリング、存在感が、サービスの約束を直接強めるとき。迷った場合の初期候補として使ってよい。

### 品質条件

- 普通のビジネス服にしない
- 表情だけに頼らず、姿勢、衣装、背景、光で意味を作る
- 顔を主役にしすぎず、タイポグラフィや色面と統合する
- 人物は構図の一部として扱う

### 避ける

- 人物なら何でもフックになると考える
- 腕組みスーツ
- 会議室で微笑む
- ただのプロフィール写真
- どの業種にも使えるストックフォト

---

## 02. Product / Object-led

### 向いている用途

- 商品LP
- 物販
- 食品
- 美容
- アプリやサービスを象徴する具体物がある場合
- 課題や変化を物で比喩化したい場合

### 設計

- 商品や象徴物を大きく、質感が伝わるように置く
- 小物は少数にする
- 影、反射、素材感を広告写真として設計する
- タイポグラフィは物の輪郭や余白に沿わせる

### 避ける

- EC商品画像のような平置き
- 小物の詰め込み
- 商品が小さい
- 何の商品かわからないほど抽象化する

---

## 03. UI / Product Screen-led

### 向いている用途

- SaaS
- AIサービス
- アプリ
- 業務改善
- 教育プラットフォーム
- ツールの価値を見せたいセクション

### 設計

- UIは1〜3面まで
- 実画面スクショではなく、価値が伝わる編集済みモックとして見せる
- 読ませるUI文字は少なくする
- 重要なカードだけ大きくし、残りは奥行きや気配として使う
- 抽象図形や色面で画面の世界観を作る

### 避ける

- ダッシュボードを細かく大量に並べる
- 小さなUI文字で説明する
- ただのプロダクトスクリーンショット
- 画面の中身がメインコピーより目立ちすぎる

---

## 04. Place / Space-led

### 向いている用途

- 宿泊
- 店舗
- 地域サービス
- イベント
- スクール
- ブランドの空気感や体験環境を見せたい場合

### 設計

- 空間そのものを主役にし、人物は必要なら小さく添える
- 光、奥行き、床や壁の質感を整理する
- 次に行きたくなる導線や余白を作る
- 文字は空間の余白に置く

### 避ける

- 不動産写真のような説明写真
- 雑然とした背景
- 暗い室内
- 観光パンフレット風の平板な写真

---

## 05. Typography-led

### 向いている用途

- 強いコピーがあるHero
- CTA
- キャンペーン
- ブランド名を記憶させたい場合
- 写真素材が弱い場合

### 設計

- 日本語コピーをロゴのように大きく設計する
- 漢字、かな、英字ラベルのジャンプ率を明確にする
- 色面、罫線、余白、少数の記号で画面を持たせる
- 写真は使っても補助に留める

### 避ける

- 文字を並べただけ
- 意味のない英字装飾
- フォント芸だけで内容がない
- 余白が足りずポスターとしてもWEBとしても弱い

---

## 06. Abstract Graphic-led

### 向いている用途

- AI
- SaaS
- コンサルティング
- 金融以外の複雑な概念
- まだ具体物がないサービス
- 変化や仕組みを視覚化したい場合

### 設計

- 抽象図形は1つの動きや構造に絞る
- 色数を抑え、アクセントを一点効かせる
- メッセージに対応する比喩を持たせる
- タイポグラフィと一体化させる

### 避ける

- 意味のないグラデーション背景
- ただの丸や波
- サイバー風の青い回路
- 何のサービスかわからない抽象アート

---

## 07. Data / Process-led

### 向いている用途

- B2B
- 業務改善
- 分析
- 比較
- Before / After
- 複雑な手順を軽く見せたい場合

### 設計

- 図解ではなく、広告的に編集されたデータ表現にする
- 数字は1〜3個まで
- 線、矢印、カード、ステップを大きく整理する
- 主メッセージとデータの読み順を一致させる

### 避ける

- 細かいチャートの大量配置
- 読めない表
- プレゼン資料のスクショ感
- データが装飾で終わっている

---

## 08. Evidence / Artifact-led

### 向いている用途

- 事例
- 実績
- 学習成果
- 制作プロセス
- 手触りや信頼の根拠を見せたい場合

### 設計

- ノート、資料、成果物、道具、手元、包装、素材などを主役にする
- 人物の顔ではなく、痕跡や成果で信頼を作る
- 細部を見せすぎず、象徴的に構成する
- 引用や数字は大きく少数にする

### 避ける

- 書類や資料を細かく読ませる
- 散らかった机
- 生活感の強すぎる手元写真
- 証拠を並べすぎて重くなる

---

## 09. Hybrid Scene-led

### 向いている用途

- 複数価値を1つの世界観にまとめたいHero
- サービスの利用シーンを豊かに見せたい場合
- 写真、UI、物、タイポグラフィを一体化させたい場合

### 設計

- 主役は1つに決め、他要素は周辺に置く
- 人物、UI、商品、抽象図形を同格にしない
- 色と光で1つの世界にまとめる
- 余白を広く取り、構成の軸を明確にする

### 避ける

- 人物、商品、UI、カード、背景を全部主役にする
- コラージュの詰め込み
- 視線の入口がない
- LP全体の要素をHeroに集める

# 選択ガード

## 人物から別モチーフへ切り替える前の確認

```text
人物写真より、そのモチーフの方がメッセージを鋭く伝えるか:
[yes/no]

人物の視覚的フックを捨てる理由があるか:
[yes/no]

商品、UI、空間、タイポグラフィ、抽象グラフィック、データ表現の方が一瞬で価値を伝えるか:
[yes/no]
```

`yes` が弱い場合は、Person-led を初期候補に戻す。別モチーフを主役にする場合も、人物の視覚的フックを失うぶん、スケール、色、光、タイポグラフィで強い入口を作る。

## セクション別の初期候補

```text
Hero: Person-led / Typography-led / Product-led / UI-led / Abstract Graphic-led
Problem: Person-led / Symbolic Object-led / Typography-led / Data Process-led / Abstract Graphic-led
Benefit: Person-led / UI-led / Abstract Graphic-led / Place-led / Product-led
Features: UI-led / Data Process-led / Product-led / Abstract Graphic-led
Case Study: Person-led / Evidence Artifact-led / Data Process-led / Place-led
Pricing: Data Process-led / Typography-led / UI-led
FAQ: Typography-led / UI-led / Abstract Graphic-led
CTA: Person-led / Typography-led / Product-led / Abstract Graphic-led
Footer: Typography-led / Logo-led / Abstract Graphic-led
```

## 品質チェック

```text
主役ビジュアルの型は1つに絞れているか:
人物を使うか、別モチーフに切り替える意図が明確か:
主役ビジュアルが主メッセージを直接支えているか:
補助要素が主役を邪魔していないか:
どの型でも広告品質の光、色、余白、タイポグラフィがあるか:
```

# references/05_color_lighting_fashion_patterns.md

## 目的

配色、光、写真、人物スタイリング、商品、UI、空間、抽象グラフィックを、WEBビジュアルとして高品質に統一するためのリファレンス。

## 共通ルール

- 明るく抜け感のある画面を優先する
- 暗い室内、曇ったグレー、沈んだトーンは禁止
- ポイントカラーを必ず入れる
- 色数は増やしすぎない
- 写真・文字・背景・装飾を同じ世界観に統一する
- 人物を主役にする場合は普通のビジネス服にしない
- 人物を使わない場合も、主役ビジュアルの光、素材感、奥行き、余白を広告品質にする
- 人物写真を強い初期候補にしつつ、商品、UI、空間、抽象グラフィックの方が強い場合は主役を切り替える
- 迷ったら、明るく、余白を取り、アクセントカラーを一点効かせる

## 配色基本構成

```text
main_color:
sub_color:
accent_color:
small_signal_color:
neutral_color:
background_tone:
```

## 推奨バランス

```text
背景・余白: 50〜70%
メインカラー: 15〜30%
サブカラー: 10〜20%
アクセントカラー: 3〜8%
小さな差し色: 1〜3%
```

アクセントカラーは面積を小さくしてもよい。重要なのは、視線を止める役割を持たせること。

# references/07_negative_examples.md

## 目的

画像生成前後のセルフチェックに使うNG例・失敗パターン・修正方針のリファレンス。

## 最重要NG

```text
WEBページ全体を1枚画像にまとめること
セクション途中で見切れること
アスペクト比に合わせて詰め込むこと
文字を小さくして無理に収めること
LP全体を縮小サムネイルのように見せること
```

---

## 01. 構造の失敗

### NG

- Hero、Problem、Benefit、Features、Pricing、FAQ、CTAを1枚に入れている
- 縦長画像にLP全体を縮小している
- セクションの途中で上下が切れている
- CTAやカードの一部が欠けている
- 見出しだけ見えて本文が切れている
- フッターや次セクションが中途半端に入っている

### 原因

- 1枚でページ全体を見せようとしている
- アスペクト比に情報を合わせようとしている
- セクション単位の分割判断が弱い

### 修正

- 1画像1セクションに戻す
- セクションをさらに分割する
- 要素を削る
- 余白を増やす
- 次セクションの要素を入れない

---

## 02. 情報過多

### NG

- カードが4枚以上ある
- 小さい説明文が大量にある
- 1画面に複数CTAがある
- 料金、特徴、事例、FAQが同居している
- 注釈やバッジが多い
- 何を見ればいいかわからない

### 原因

- 全部伝えようとしている
- 情報の優先順位がない
- セクションの主メッセージが1つに絞れていない

### 修正

- 主メッセージを1つにする
- カードは最大3枚
- 詳細は別セクションへ分ける
- コピーを短くする
- CTAを1つにする

---

## 03. タイポグラフィの失敗

### NG

- 白文字を写真の上に置いただけ
- 既成フォントをベタ打ちしただけ
- 全文字が同じサイズ
- 改行が不自然
- 日本語が読めない
- 意味不明な英字
- 助詞まで大きくて野暮ったい
- 文字が小さすぎてスマホで読めない

### 原因

- 文字を情報としてしか扱っていない
- 日本語のサイズ差・余白・改行を設計していない
- タイポグラフィ型を選んでいない

### 修正

- 極太ゴシックロックアップを使う
- 漢字巨大化 × かな抑制を使う
- 字間広めのサブコピーを使う
- メインコピーを短くする
- 読ませる文字を欠けさせない
- 英字は意味のある短いラベルだけにする

---

## 04. 写真の失敗

### NG

- 安いストックフォト風
- 普通のビジネス服
- 会議室で笑顔
- 腕組みスーツ
- 顔や手が不自然
- 肌がくすんでいる
- 暗い室内
- 人物が商品や文字と分離している

### 原因

- 写真の世界観設計がない
- 光・衣装・背景が凡庸
- 広告写真ではなく説明写真になっている

### 修正

- ファッションポートレート化する
- 明るい自然光またはハイキー光を使う
- 衣装にアクセントカラーを入れる
- 背景と文字色を連動させる
- 人物をただ置かず、構図の一部にする

---

## 05. 配色の失敗

### NG

- 色数が多すぎる
- アクセントカラーがない
- 画面全体がグレーで沈む
- 原色が多く下品
- 背景と文字のコントラスト不足
- 写真と文字の色が合っていない

### 原因

- 配色の役割分担がない
- メイン、サブ、アクセント、ニュートラルを決めていない
- アクセントの面積が大きすぎる

### 修正

- メインカラー1色
- サブカラー1色
- アクセントカラー1色
- ニュートラル1〜2色
- アクセントは小さく効かせる
- 背景は明るく抜け感を出す

---

## 06. トーンの失敗

### NG

- Canvaテンプレート風
- 情報商材バナー風
- 美容クリニック広告風
- 古いチラシ風
- 地方イベント告知風
- 怪しい副業広告風
- 無難すぎて記憶に残らない
- おしゃれだが何のサービスかわからない

### 原因

- 業種別のありがちな記号に頼っている
- 広告目的とビジュアルがつながっていない
- 型を選ばず、雰囲気だけで作っている

### 修正

- 業種別トーンを参照する
- メインメッセージを明確にする
- 1つのグラフィック構図型を選ぶ
- CTAとのつながりを確認する

---

## 07. AI画像らしさ

### NG

- 手や顔が破綻
- 文字が崩れている
- UI画面の文字が意味不明
- 商品ロゴが不自然
- 背景の構造が曖昧
- 人物のポーズが不自然
- 写真の質感がプラスチックっぽい

### 原因

- 生成対象が複雑すぎる
- 文字情報を入れすぎている
- 人物やUIを細かく指定しすぎている

### 修正

- 画像内テキストを減らす
- UIは雰囲気重視にする
- 顔や手が目立つポーズを避ける
- 主役を1つに絞る
- 重要な文字は大きく短くする

---

## 08. WEBらしさの不足

### NG

- ポスターとしては良いがWEBセクションに見えない
- CTAがない
- 次の行動がわからない
- セクションの役割が不明
- 他セクションと並べたときにつながらない

### 原因

- グラフィック表現が強すぎる
- WEBの情報設計が弱い
- セクションの役割を決めていない

### 修正

- section_roleを明確にする
- 必要なCTAを入れる
- WEBレイアウト型を選ぶ
- 次セクションとの連続性を持たせる
- ポスター感とWEB構造を両立する

---

## 09. 主役ビジュアル選択の失敗

### NG

- どのHeroも人物写真が主役になっている
- 人物写真を使っているが、スタイリングや構図が凡庸で視覚的フックになっていない
- 商品、UI、空間、タイポグラフィ、抽象グラフィックの方が伝わるのに人物を置いている
- 人物を避けること自体が目的になり、別モチーフの画面が弱くなっている
- 人物、商品、UI、カード、背景が全部同じ強さで並んでいる
- 主役ビジュアルがメインコピーの意味を支えていない
- 抽象グラフィックに逃げた結果、何のサービスかわからない

### 原因

- 主役ビジュアルの型を選んでいない
- 人物写真なら自動的にフックになると誤解している
- 別モチーフに切り替える理由を決めていない
- メッセージから逆算せず、雰囲気で素材を置いている
- 複数の型を同時に主役化している

### 修正

- `09_main_visual_archetypes.md` から主役ビジュアルの型を1つ選ぶ
- 迷ったら Person-led を初期候補にする
- 人物より商品、UI、空間、象徴物、タイポグラフィ、抽象グラフィック、データ表現の方が強い場合だけ主役を切り替える
- 補助要素を減らし、主役を大きく見せる
- 主役ビジュアルとメインコピーの意味を一致させる

---

## 修正コマンド集

### 詰め込みすぎた場合

```text
情報を半分に減らす。
この画像は1セクションだけにする。
読ませるコピーを1つに絞る。
詳細は次のセクション画像へ分割する。
余白を広く取り、カード数を最大3枚にする。
```

### 見切れている場合

```text
セクション全体が自然に完結して見えるように再構成する。
見出し、本文、CTA、人物、商品、カードを欠けさせない。
画面端で切ってよいのは装飾的な背景要素のみ。
途中で切り抜かれたスクリーンショットに見えないようにする。
```

### 安っぽい場合

```text
Canvaテンプレート風を避ける。
情報商材バナー風を避ける。
文字数を減らし、ジャンプ率を上げる。
余白を増やし、写真とタイポグラフィを広告品質にする。
ポイントカラーを一点だけ効かせる。
```

### タイポグラフィが弱い場合

```text
日本語コピーをロゴのように再設計する。
漢字を大きく、かなや助詞を小さくする。
メインコピーのジャンプ率を上げる。
白文字ベタ打ちを避ける。
字間・行間・改行位置を広告的に設計する。
```

### 人物写真が凡庸な場合 / 別モチーフの方が強い場合

```text
人物写真は強い初期候補として扱う。
人物の表情、姿勢、衣装、背景、光が弱い場合は、人物写真そのものを広告品質に上げる。
商品、UI、空間、象徴物、タイポグラフィ、抽象グラフィック、データ表現の方が一瞬で価値を伝えられる場合は、意図的に主役を切り替える。
主役以外の要素を減らし、余白とアクセントカラーで広告品質にする。
```

```text
<generate_image section_id="section_04_features">

目的:
サービスや商品の選ばれる理由を整理して見せるFeatures Section。

主メッセージ:
[最も重要な特徴または選ばれる理由]

画面内テキスト:
- セクションラベル: FEATURES
- メインコピー: [short_feature_copy]
- 特徴: [最大3点]
- 補足: [必要なら短く]

主役ビジュアル:
UIカード、商品、アイコンではなく象徴的なグラフィック、機能を表すビジュアル。

デザイン:
- Card Stack Layout / Editorial Grid / Layered UI
- 主カードを1つ大きく、補助カードは控えめに
- 日本語タイポグラフィは字間広めのサブコピー、欧文ラベル添え、長体ゴシック
- 余白を保つ
- 図解ではなく広告的に見せる

制約:
- 6項目以上を詰め込まない
- すべての機能を説明しない
- UIカードを小さく大量配置しない
- 入りきらない場合は Feature 01 / Feature 02 に分割
- 1画像1セクション

</generate_image>
```

## 13. Revision Prompt Template

```text
<revise_image section_id="[section_id]">

現在の問題:
[見切れ / 詰め込み / 文字が小さい / 安っぽい / 暗い / タイポグラフィが弱い / AI破綻 / その他]

修正方針:
- 1画像1セクションを維持する
- セクション全体が自然に完結して見えるようにする
- 見出し、本文、CTA、人物、商品、カードを欠けさせない
- 情報を減らす
- 余白を増やす
- メインコピーを短くする
- 日本語タイポグラフィのジャンプ率を上げる
- ポイントカラーを一点だけ効かせる
- 明るく抜け感のある光にする
- 人物写真が凡庸な場合、または別モチーフの方が価値を一瞬で伝える場合は、主役ビジュアル型を選び直す
- 安っぽいテンプレート感を消す

禁止:
- LP全体を1枚にまとめる
- セクション途中でトリミングする
- 文字を小さくして詰め込む
- 複数メッセージを混ぜる

</revise_image>
```
