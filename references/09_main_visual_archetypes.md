# references/09_main_visual_archetypes.md

## 目的

WEBセクションの主役ビジュアルを、人物写真を強い既定候補として扱いながら、必要に応じて別モチーフへ意図的に切り替えるためのリファレンス。

主役ビジュアルの型は、クオリティを分散させるためではなく、**そのセクションの一番強い意味を何で見せるか**を決めるために使う。

## 最重要ルール

- 各セクションで主役ビジュアルの型を1つだけ選ぶ
- 人物写真は強い既定候補として扱う
- 特にHero、Benefit、CTA、採用、教育、個人ブランド、体験型サービスでは Person-led を初期候補にする
- 別モチーフは、人物よりも商品、UI、空間、物、抽象グラフィック、タイポグラフィ、データ表現の方がメッセージを鋭く見せられる場合に意図的に選ぶ
- 人物を使わないこと自体を目的にしない
- 複数の型を同格に並べない
- 補助ビジュアルは主役の意味を支えるだけにする
- 主役は大きく、明確に、余白を持って配置する
- どの型でも、広告品質の光、色、余白、タイポグラフィを必ず設計する

## 選択順

```text
1. このセクションの主メッセージは何か
2. まず Person-led を初期候補に置く
3. そのメッセージは、人・物・画面・空間・構造・言葉のどれで最も強く伝わるか
4. 人物より強い理由がある場合だけ、別モチーフへ切り替える
5. 主役ビジュアルの型を1つ選ぶ
6. 色・光・タイポグラフィを、その型に合わせて統一する
```

迷った場合は、Person-led を初期候補にする。ただし、商品・UI・空間・タイポグラフィ・抽象グラフィックの方が一瞬で価値を伝えられる場合は、別モチーフを主役にする。

# 主役ビジュアルの型

## 01. Person-led

### 向いている用途

- 個人ブランド
- 採用
- スクール
- コーチング
- 体験型サービス
- 顧客の感情変化を見せたいセクション
- Hero
- Benefit
- CTA

### 使う条件

人物の表情、姿勢、スタイリング、存在感が、サービスの約束を直接強めるとき。迷った場合の初期候補として使ってよい。

### 品質条件

- 普通のビジネス服にしない
- 表情だけに頼らず、姿勢、衣装、背景、光で意味を作る
- 顔を主役にしすぎず、タイポグラフィや色面と統合する
- 人物は構図の一部として扱う

### 避ける

- 人物なら何でもフックになると考える
- 腕組みスーツ
- 会議室で微笑む
- ただのプロフィール写真
- どの業種にも使えるストックフォト

---

## 02. Product / Object-led

### 向いている用途

- 商品LP
- 物販
- 食品
- 美容
- アプリやサービスを象徴する具体物がある場合
- 課題や変化を物で比喩化したい場合

### 設計

- 商品や象徴物を大きく、質感が伝わるように置く
- 小物は少数にする
- 影、反射、素材感を広告写真として設計する
- タイポグラフィは物の輪郭や余白に沿わせる

### 避ける

- EC商品画像のような平置き
- 小物の詰め込み
- 商品が小さい
- 何の商品かわからないほど抽象化する

---

## 03. UI / Product Screen-led

### 向いている用途

- SaaS
- AIサービス
- アプリ
- 業務改善
- 教育プラットフォーム
- ツールの価値を見せたいセクション

### 設計

- UIは1〜3面まで
- 実画面スクショではなく、価値が伝わる編集済みモックとして見せる
- 読ませるUI文字は少なくする
- 重要なカードだけ大きくし、残りは奥行きや気配として使う
- 抽象図形や色面で画面の世界観を作る

### 避ける

- ダッシュボードを細かく大量に並べる
- 小さなUI文字で説明する
- ただのプロダクトスクリーンショット
- 画面の中身がメインコピーより目立ちすぎる

---

## 04. Place / Space-led

### 向いている用途

- 宿泊
- 店舗
- 地域サービス
- イベント
- スクール
- ブランドの空気感や体験環境を見せたい場合

### 設計

- 空間そのものを主役にし、人物は必要なら小さく添える
- 光、奥行き、床や壁の質感を整理する
- 次に行きたくなる導線や余白を作る
- 文字は空間の余白に置く

### 避ける

- 不動産写真のような説明写真
- 雑然とした背景
- 暗い室内
- 観光パンフレット風の平板な写真

---

## 05. Typography-led

### 向いている用途

- 強いコピーがあるHero
- CTA
- キャンペーン
- ブランド名を記憶させたい場合
- 写真素材が弱い場合

### 設計

- 日本語コピーをロゴのように大きく設計する
- 漢字、かな、英字ラベルのジャンプ率を明確にする
- 色面、罫線、余白、少数の記号で画面を持たせる
- 写真は使っても補助に留める

### 避ける

- 文字を並べただけ
- 意味のない英字装飾
- フォント芸だけで内容がない
- 余白が足りずポスターとしてもWEBとしても弱い

---

## 06. Abstract Graphic-led

### 向いている用途

- AI
- SaaS
- コンサルティング
- 金融以外の複雑な概念
- まだ具体物がないサービス
- 変化や仕組みを視覚化したい場合

### 設計

- 抽象図形は1つの動きや構造に絞る
- 色数を抑え、アクセントを一点効かせる
- メッセージに対応する比喩を持たせる
- タイポグラフィと一体化させる

### 避ける

- 意味のないグラデーション背景
- ただの丸や波
- サイバー風の青い回路
- 何のサービスかわからない抽象アート

---

## 07. Data / Process-led

### 向いている用途

- B2B
- 業務改善
- 分析
- 比較
- Before / After
- 複雑な手順を軽く見せたい場合

### 設計

- 図解ではなく、広告的に編集されたデータ表現にする
- 数字は1〜3個まで
- 線、矢印、カード、ステップを大きく整理する
- 主メッセージとデータの読み順を一致させる

### 避ける

- 細かいチャートの大量配置
- 読めない表
- プレゼン資料のスクショ感
- データが装飾で終わっている

---

## 08. Evidence / Artifact-led

### 向いている用途

- 事例
- 実績
- 学習成果
- 制作プロセス
- 手触りや信頼の根拠を見せたい場合

### 設計

- ノート、資料、成果物、道具、手元、包装、素材などを主役にする
- 人物の顔ではなく、痕跡や成果で信頼を作る
- 細部を見せすぎず、象徴的に構成する
- 引用や数字は大きく少数にする

### 避ける

- 書類や資料を細かく読ませる
- 散らかった机
- 生活感の強すぎる手元写真
- 証拠を並べすぎて重くなる

---

## 09. Hybrid Scene-led

### 向いている用途

- 複数価値を1つの世界観にまとめたいHero
- サービスの利用シーンを豊かに見せたい場合
- 写真、UI、物、タイポグラフィを一体化させたい場合

### 設計

- 主役は1つに決め、他要素は周辺に置く
- 人物、UI、商品、抽象図形を同格にしない
- 色と光で1つの世界にまとめる
- 余白を広く取り、構成の軸を明確にする

### 避ける

- 人物、商品、UI、カード、背景を全部主役にする
- コラージュの詰め込み
- 視線の入口がない
- LP全体の要素をHeroに集める

# 選択ガード

## 人物から別モチーフへ切り替える前の確認

```text
人物写真より、そのモチーフの方がメッセージを鋭く伝えるか:
[yes/no]

人物の視覚的フックを捨てる理由があるか:
[yes/no]

商品、UI、空間、タイポグラフィ、抽象グラフィック、データ表現の方が一瞬で価値を伝えるか:
[yes/no]
```

`yes` が弱い場合は、Person-led を初期候補に戻す。別モチーフを主役にする場合も、人物の視覚的フックを失うぶん、スケール、色、光、タイポグラフィで強い入口を作る。

## セクション別の初期候補

```text
Hero: Person-led / Typography-led / Product-led / UI-led / Abstract Graphic-led
Problem: Person-led / Symbolic Object-led / Typography-led / Data Process-led / Abstract Graphic-led
Benefit: Person-led / UI-led / Abstract Graphic-led / Place-led / Product-led
Features: UI-led / Data Process-led / Product-led / Abstract Graphic-led
Case Study: Person-led / Evidence Artifact-led / Data Process-led / Place-led
Pricing: Data Process-led / Typography-led / UI-led
FAQ: Typography-led / UI-led / Abstract Graphic-led
CTA: Person-led / Typography-led / Product-led / Abstract Graphic-led
Footer: Typography-led / Logo-led / Abstract Graphic-led
```

## 品質チェック

```text
主役ビジュアルの型は1つに絞れているか:
人物を使うか、別モチーフに切り替える意図が明確か:
主役ビジュアルが主メッセージを直接支えているか:
補助要素が主役を邪魔していないか:
どの型でも広告品質の光、色、余白、タイポグラフィがあるか:
```
